📊 RappiPlus Analytics

Diagnóstico Estratégico Integral: De Datos a Decisiones de Negocio

📌 Descripción del Proyecto

Este proyecto simula el trabajo de un Data Analyst dentro de una empresa de delivery llamada RappiPlus. El objetivo es transformar datos provenientes de ventas, marketing, comportamiento de usuarios y experimentos A/B en información accionable para mejorar la rentabilidad, optimizar el embudo de conversión y aumentar la retención de clientes.

El proyecto integra limpieza de datos, análisis exploratorio, KPIs de negocio, SQL, cohortes, pruebas estadísticas A/B y storytelling con dashboards.

🎯 Objetivos del Proyecto

¿El negocio es rentable?
¿Qué productos generan mayor utilidad?
¿Dónde abandonan los usuarios?
¿Los clientes regresan después de registrarse?
¿La nueva interfaz mejora la conversión?
¿Qué debe visualizar la dirección para decidir mejor?

📂 Fuentes de Datos

Orders

Información de pedidos: fecha, producto, cantidad, precio, revenue, costo y profit.

Catálogo

Información de productos: categoría, proveedor y costo unitario.

Marketing

Inversión publicitaria por canal, país y gasto.

Eventos

Comportamiento de usuarios dentro de la app: Home, Search, Product, Cart, Checkout y Purchase.

Usuarios

Información utilizada para calcular retención y cohortes.

Experimento A/B

Comparación entre grupo Control y Tratamiento para evaluar la nueva interfaz de checkout.

📈 Desarrollo del Proyecto

1️⃣ Calidad de Datos

Conversión de fechas, revisión de nulos, eliminación de duplicados, corrección de valores negativos, validación de montos y estandarización de categorías.

2️⃣ Rentabilidad

Cálculo de revenue total, costos, profit, margen, ticket promedio, productos más vendidos y categorías más rentables.

3️⃣ Funnel de Conversión

Análisis con SQL para detectar en qué etapas del recorrido los usuarios abandonan la aplicación.

4️⃣ Cohortes

Medición de usuarios recurrentes, tasa de retención, semanas con mayor abandono y comportamiento por cohorte.

5️⃣ Prueba A/B

Evaluación estadística de la nueva interfaz de checkout mediante diferencia de proporciones entre Control y Tratamiento.

6️⃣ Dashboard Ejecutivo

Diseño de visualizaciones para comunicar revenue, profit, costos, marketing, conversión, retención y principales KPIs.

🧪 Prueba A/B

Se evaluó una nueva versión del checkout comparando el comportamiento de usuarios del grupo Control contra el grupo Tratamiento. El objetivo fue determinar si la nueva interfaz incrementa la conversión de forma estadísticamente significativa.

La decisión se basa en evidencia estadística, no solo en diferencias observadas entre porcentajes.

📊 Dashboard Ejecutivo

El dashboard está diseñado para facilitar la toma de decisiones de negocio, mostrando métricas clave de rentabilidad, conversión, retención y desempeño comercial.
